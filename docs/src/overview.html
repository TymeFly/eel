This manual describes the standard EEL $eel.version() functions that are automatically available to all expressions without needing to explicitly add them via the EEL Context.

Many EEL functions can accept a variable number of arguments. The final argument can appear zero, one, or multiple times. This is indicated in the list below by ... after the last argument. For example, date.plus(date, offsets...) requires a date but can accept any number of offsets, including none.

Some functions support default arguments. If a value is not explicitly provided, EEL will automatically use the default. When a parameter can be defaulted, this documentation will describe the default value. For example, the random function can be called as random(), random(min), or random(min, max), where min defaults to 0 and max defaults to 100.

As EEL is a lazy language, parameters are evaluated only when they are used. Consequently, functions such as count(), fail(), and the logging functions can be safely passed as parameters, as there will be no side effects if they are not used.
