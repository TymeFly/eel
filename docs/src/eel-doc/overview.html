This manual describes the standard EEL $eel.version() functions that are automatically available in all expressions without needing to add them explicitly via the EEL context.

Many EEL functions accept a variable number of arguments. The final argument can appear zero, one, or multiple times. This is indicated by ... after the last argument. For example:

  • date.plus(date, offsets...) requires a date but can accept any number of offsets, including none.

Some functions support default arguments. If a value is not explicitly provided, EEL automatically uses the default. When a parameter can be defaulted, this documentation specifies the default value. For example:

  • random(), for a random number between 0 and 99 inclusive
  • random(min), for a random number between min and 99 inclusive
  • random(min, max), for a random number between min and max inclusive

As EEL is a lazy language, parameters are evaluated only when used. Consequently, functions such as count(), fail(), and the logging functions can safely be passed as parameters without side effects if they are not used.
