date

Functions that manipulate or generate dates.
Most date functions support modifying, setting, offsetting, or rounding a date with respect to one or more time periods. The supported time periods are:

PeriodFull NamesShort NameRangeNotes
Yearsyear, yearsy-999,999,999 to 999,999,999
Monthsmonth, monthsM 1 to 12
Weeksweek, weeksw 0 to 54 [1]
Daysday, daysd 1 to 28, 29, 30 or 31
Hourshour, hoursh 0 to 23
Minutesminute, minutesm 0 to 59
Secondssecond, secondss 0 to 59
Millimilli, millisI 0 to 999 [2]
Micromicro, microsU 0 to 999 [2]
Nanonano, nanosN 0 to 999 [2]
Milli of secondmilliOfSecond, millisOfSecondi 0 to 999 [3]
Micro of secondmicroOfSecond, microsOfSecondu 0 to 999,999 [3]
Nano of secondnanoOfSecond, nanosOfSecondn 0 to 999,999,999 [3]

Notes:

  1. The first day of a week and the first week of a year are set in the EEL context. By default, the first day of the week is Monday, and there are at least 4 days in the first week, following ISO standard 8601.
  2. Milli, Micro, and Nano are independent. For example, setting microseconds does not affect milliseconds or nanoseconds.
  3. Milli of Second, Micro of Second, and Nano of Second update all fractional parts of a second. For example, setting microseconds to 1,234 sets millis to 1, micros to 234, and nanos to 0. For adding, subtracting, or snapping, these periods act like their independent counterparts.

Period names are case-sensitive. Time zones can be:

  • Fixed offsets – a resolved offset from UTC such as +5
  • Geographical regions – a region following specific rules to find offsets, e.g., Europe/Paris

Time periods can be snapped to the start of a period with a modifier @<period>. Snap modifiers always round down.

Examples of valid modifiers:

  • 7d : 7 days in the future
  • -7d : 7 days in the past
  • 7days : 7 days in the future using the full name
  • @d : the start of the current day

Offsets are applied in the order passed. Examples:

  • date.utc() – current UTC date-time with nanosecond accuracy
  • date.utc("@s") – current UTC date-time with second accuracy
  • date.utc("1day") – current UTC date-time plus 1 day
  • date.utc("@d", "+12h") – current date at midday UTC
  • date.utc("@d", "-1_500n") – 1500 nanoseconds before start of UTC date
  • date.utc("1d", "@d") – midnight tomorrow UTC
  • date.utc("48h", "@d") – midnight the day after tomorrow UTC
  • date.start("Europe/Paris") – instant the EEL context was created in Paris
  • date.local("+2months") – current local date-time plus 2 months
  • date.at("-5", "-1w") – current date-time in UTC-5 minus 1 week
  • date.set(${value}, "12h", "0m", "0s", "0n") – given date with fields set to midday
  • date.plus(${value}, "15minutes") – given date plus 15 minutes
  • date.minus(${value}, "-15minutes") – given date minus 15 minutes
  • format.at("Europe/Paris", "d/M/yyyy HH:mm") – current Paris date and time in given format

date.moveZone(date, zone)

Returns a copy of the given date with a different time zone. Unlike date.setZone(date, zone), this function adjusts the time fields so that the numeric value of the returned date (seconds since 1970-01-01 00:00:00 UTC) matches the original date.

Called as:

date.moveZone(date, zone)

Parameters:

date
Type : Date
Description : The date to update.
zone
Type : Text
Description : The zone ID.

Return type:

Date

Returns:

The same instant as the given date but in a different time zone.

Throws:

DateTimeException If the zone is not valid.

See:

date.setZone(date, zone)

Returns a copy of the given date with a different time zone. Unlike date.moveZone(date, zone), this function preserves the time fields. The numeric value of the returned date (seconds since 1970-01-01 00:00:00 UTC) may therefore differ from the original date.

Called as:

date.start(zone, offsets...)

Returns the DATE at which the given context was created in the UTC time zone. This provides a timestamp that does not change across repeated invocations of an expression or related expressions that use the same context.

Called as:

date.start()
date.start(zone)
date.start(zone, offsets...)

Parameters:

zone
Type : Text
Default Value : UTC
Description : The time zone name.
offsets...
Type : Text
Default Value : No offsets
Description : Offsets to apply.

Return type:

Date

Returns:

A timestamp indicating when the context was created.

Throws:

DateTimeException If the zone is not valid.
