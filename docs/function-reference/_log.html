log

Writes messages to the system logs. The value returned by the function is the last argument passed. Logging functions, in order of priority from highest to lowest, are:
  • log.error(first, others...)
  • log.warn(first, others...)
  • log.info(first, others...)
  • log.debug(first, others...)
  • log.trace(first, others...)
At least one argument must be passed to the logging function.

Logging can be used by the client to debug problematic expressions or monitor values generated by an expression. For example: 

 $log.debug(1 + 2 + 3)
calculates 6, logs it at debug level, and returns it as NUMBER to the client application. Logging hardcoded values is rarely useful; however, EEL can log values read from the symbols table. For example: 
 Hello: $log.info(${userName})
reads a value from the symbols table, logs it, and uses it as part of a friendly message.

EEL also supports conditional logging. For example: 

 The Result is: ${key-$log.info("defaulting to {}", "defaultValue")}
If key exists in the symbols table, its value is returned without logging. If the value cannot be read, a warning is logged and defaultValue is returned. This works because a variable's default value is an EEL expression and can contain embedded function calls.

Notes:

  1. The client's logging framework is responsible for enabling logging.
  2. All messages are written to the com.github.tymefly.eel.log logger, which should be configured with trace level logging enabled.
  3. To prevent log repudiation attacks:
    • All logged messages are prefixed with the literal text Logged EEL Message: to indicate they are generated by EEL. This prefix cannot be changed or removed.
    • Only printable characters and tabs are logged. Control characters, including new lines and Unicode characters outside the printable range, are silently filtered out.
